Chimpan's Logger - Manual

1.概要・処理フロー

スマートフォンのメモ帳等で記録したポーカーハンドのテキストメモ（.txt）を読み込み、After Effects 用の scenes.json を自動生成するツールです。

memo.txtテキストメモ
→
poker_memo_converter.pyPython 変換
→
scenes.jsonAE 用データ
→
poker_ae_builder.jsxAfter Effects
ポイント: メモは時系列順にアクションを書くだけ。ポジション名・ポット計算・イベント構築はすべて自動処理されます。

2.使い方

方法 A：ドラッグ＆ドロップ（推奨）

  1. .txt ファイルを memo2json.bat のアイコンにドラッグ＆ドロップ
  2. コマンドプロンプトが開き、変換が実行される
  3. 成功すると .txt と同じフォルダ に hand##_scenes.json が生成される

bat ファイルと .txt は別フォルダにあっても動作します。出力先は常に .txt 側のフォルダです。

方法 B：コマンドライン

# 基本
python poker_memo_converter.py hand03_memo.txt

# 出力先を指定
python poker_memo_converter.py hand03_memo.txt -o output/

方法 C：Memo Builder で作成 → 保存 → 変換

  1. memo_template.html をブラウザで開く
  2. ヘッダー情報・カードを入力し、テンプレート生成
  3. テキストエリア内でアクションを追記・編集
  4. 「Save (.txt)」で保存、bat にドロップして変換
前提条件: Python 3 がインストールされている必要があります。外部ライブラリは不要（標準ライブラリのみ使用）。
Windows 互換性: bat ファイルは純粋 ASCII（改行 CRLF）で記述されており、どの Windows 環境でも動作します。Python 側は sys.stdout.reconfigure(encoding='utf-8') により文字化けしません。

3.メモの書き方 — 全体構造

メモは上から順に以下のセクションで構成されます。

# ① ヘッダー（メタ情報）
#03
sb200 bb400 ante400
hero 10 87200 9cJc
8max
video ファイル名.mp4
ifb assets/interface/base.png
ifi assets/interface/info.png

# ② プレイヤー一覧（UTG順）
2 プレイヤーA
3 プレイヤーB
...

# ③ ストリート（アクション）
---pf
---flop 6d 7s 3h
---turn 5s
---river 2s

# ④ ショーダウン（任意）
---sd

# ⑤ リザルト
---result
ルール:
  • 空行は無視されます
  • セクション区切りは --- で始まる行です
  • 行頭・行末の余白（スペース・タブ）は自動トリムされます
  • # の後にスペース＋文字が続く行はコメントとして無視されます（例: # メモ書き）。
    ハンド番号の #03 とは区別されます（# の直後が数字 → ハンド番号）

4.ヘッダー（メタ情報）

ファイル先頭〜最初のプレイヤー行までがヘッダーです。順序は自由です。

項目書式例
ハンド番号 必須#番号#03
ブラインド 必須sb金額 bb金額sb200 bb400
アンティ 任意ante金額（sb行に追記も可）ante400 / sb200 bb400 ante400
ヒーロー 必須hero シート スタック カードhero 10 87200 9cJc
テーブル人数 必須数字max8max
動画パス 任意video パスvideo hand03_base.mp4
IFベース 任意ifb パスifb assets/interface/base.png
IF情報 任意ifi パスifi assets/interface/info.png
ヒーローのカード: 9cJc のように繋げても、9c Jc のようにスペースで区切っても認識されます。10を含む場合（例: 10hAs）も対応しています。

5.プレイヤー一覧

ヘッダーの後、---pf の前に記述します。UTG から時計回り順に記載してください。

# シート番号 プレイヤー名
2 プレイヤーA        ← UTG（1番目に書いた人）
3 プレイヤーB        ← +1（2番目）
4 プレイヤーC        ← LJ（3番目）
6 白Tニキ           ← HJ（4番目）
8 プレイヤーD        ← CO（5番目）
9 プレイヤーE        ← BTN（6番目）
10 チンパン          ← SB（7番目）
1 プレイヤーF        ← BB（8番目）
重要:
  • シート番号は卓上の実際の座席番号（1〜10）
  • ヒーローも必ずプレイヤー一覧に含めてください
  • ポジション名は入力不要 — Nmax と並び順から自動で割り当てられます
  • シート番号が連番でなくても問題ありません（欠番OK）

6.ポジション自動割り当て

Nmax で指定した人数のポジションテーブルが選択され、UTG から順に割り当てられます。

10max
BTN SB BB UTG +1 MP1 MP2 LJ HJ CO
9max
BTN SB BB UTG +1 MP LJ HJ CO
8max
BTN SB BB UTG +1 LJ HJ CO
7max
BTN SB BB UTG +1 HJ CO
6max
BTN SB BB UTG +1 CO
5max
BTN SB BB UTG CO
4max
BTN SB BB UTG
3max
BTN SB BB
2max
BTN BB

プレイヤー一覧は UTG を起点にローテーションして割り当てられます。

例（8max）: テーブルには UTG +1 LJ HJ CO BTN SB BB の8ポジション。一覧の1人目 → UTG、2人目 → +1 … 8人目 → BB と順番に対応します。

7.アクション省略記法
記法アクション金額例
f → fold金額不要3 f
x → check金額不要10 x
c金額 → call金額 必須6 c2000
b金額 → bet金額 必須7 b3500
r金額 → raise金額 必須3 r8000
ai金額 → allin金額 必須10 ai24300
str金額 → straddle金額 必須4 str800

各行は シート番号 アクション の形式で、発生順（時系列順）に記述します。

金額に注意: c b r ai str には金額が必須です。金額なしはバリデーションエラーになります。f と x には金額は不要です。

9.ショーダウン & リザルト

ショーダウン ---sd

最終ストリートの後、カードを見せた（またはマックした）プレイヤーを記載します。

書式意味例
シート カード1 カード2ハンドを公開7 Ah 7h
シート showhero公開（カード自動取得）10 show
シート muckカードを伏せる2 muck
ショーダウンタイミングの自動判定:
• カードを公開したプレイヤーが 2人以上 → river タイミング
• カードを公開したプレイヤーが 1人 → result タイミング

リザルト ---result

勝者を記載します。

書式意味例
シート win単独勝利10 win
シート chop（複数行）引き分け4 chop / 7 chop
注意:
  • win は1人だけ指定可能です（複数人はエラー）
  • 引き分けの場合は全員に chop を使用してください
  • win と chop の混在はエラーになります
  • フォールド済みのプレイヤーを勝者に指定するとエラーになります

10.カード表記ルール
ランク表記
2〜92 3 4 5 6 7 8 9
10T または 10
絵札・エースJ Q K A
スート表記
スペードs
ハートh
ダイヤd
クラブc

組み合わせ例: Ah（エースのハート）、10c（10のクラブ）、Ts（10のスペード）、2d（2のダイヤ）

大文字・小文字: ランクは大文字推奨（A K Q J T）、スートは小文字推奨（s h d c）ですが、混在しても認識されます。

11.バリデーション（自動検閲）

変換実行時に自動でメモの整合性がチェックされます。

エラー（致命的） — 変換中止

チェック内容説明
必須項目の欠落ハンド番号 / SB / BB / hero / Nmax / プレイヤー2人以上 / ヒーローカード
ヒーローがプレイヤー一覧にいないhero のシート番号が一覧に含まれているか
不正なカード表記ランク・スートの組み合わせが正しいか
カード重複全カード（hero / board / showdown）の中に同じカードがないか
フロップ枚数フロップのボードカードが3枚か
ストリート順序ターンがあるのにフロップがない等
存在しないシート番号アクション・ショーダウン・リザルトで指定されたシートが一覧にいるか
金額の欠落call / bet / raise / allin / straddle に金額が指定されているか
プリフロップにアクションなし---pf セクションが空
ショーダウンにフォールド済み既にフォールドしたプレイヤーがショーダウンにいないか
リザルト不正win/chop以外の結果、win複数指定、win+chop混在、フォールド済み勝者
リザルト欠落---result セクション自体がない

警告（注意） — 変換は続行

チェック内容説明
ヒーロースタック0 または未設定
Nmax とプレイヤー数の不一致例: 8max なのに 7 人
パス未設定video / ifb / ifi が空
ボードありアクションなしフロップ等にボードがあるのにアクションが記載されていない
chop が1人だけ引き分けは通常2人以上
アクション順序不正ストリート別の正しいアクション順序と異なる（プリフロップ: UTG→BB、ポストフロップ: SB→BTN）
⚠ アクション順序の確認プロンプト
アクション順序に問題がある場合、変換時に「このまま続行しますか？ (y/N)」と確認が表示されます。
意図的に通常と異なる順序で記載している場合は y で続行できます。

出力例

  [hand03_memo.txt]
  --- 警告 (1) ---
  [!] 8max ですがプレイヤーが 7 人です
  [OK] D:\...\hand03_scenes.json

  [hand03_memo.txt]
  --- エラー (2) ---
  [X] ヒーローのカードが未設定です
  [X] preflop: seat 99 はプレイヤー一覧にいません

  [STOP] エラーがあるため変換を中止しました
         メモを修正して再実行してください

13.Memo Builder（テンプレート生成ツール）

memo_template.html をブラウザで開くと、タップ操作でメモのテンプレートを作成できます。iPhone / Android / Windows / Mac すべてで動作します（オフライン対応・依存なし）。

機能一覧

機能説明
ヘッダー入力Hand番号, SB, BB, Ante, Seat, Stack, Nmax を1画面で入力
カードピッカーランクタップ → スートタップの2タップで選択。使用済みカードは全ピッカーで共有・自動グレーアウト
プレイヤーリスト席番号と名前を入力。行の追加・削除が可能
ボードカードFlop(3枚) / Turn(1枚) / River(1枚) をタップで選択（任意）
パス設定video / ifb / ifi を任意入力（折りたたみ式）
テンプレート生成入力内容からメモテキストを自動生成
編集生成後のテキストエリアは編集可能。アクションをその場で追記できる
出力Copy（クリップボード）/ Save (.txt) / Share（モバイル共有API）

基本フロー

memo_template.html情報入力
→
テンプレート生成テキスト出力
→
アクション追記画面内で編集
→
Save / Copy.txt保存
対応環境:
  • iPhone / Android — Copy + Share ボタン（共有API）
  • Windows / Mac — Copy + Save (.txt) ボタン（Blobダウンロード）
  • file:// プロトコルでも動作（ダブルクリックで開くだけでOK）
